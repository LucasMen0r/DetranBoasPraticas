CATEGORIA: Stored Procedures
========================================
PERGUNTA: Como devo nomear uma procedure que atualiza dados de veículos roubados no DETRAN-PE?
========================================
RESPOSTA:
Procede com base no padrão Objetivo[Complemento]Operação. Use 'VeiculoRoubado' como objetivo e 'Atualizar' como complemento, terminando com o sufixo 'E' para Excluir. Exemplo: AtualizarVeiculoRoubadoExcluirE.scp (Regra: Operação E para exclusão).

CATEGORIA: Nomenclatura de Objetos
========================================
PERGUNTA: Posso usar o prefixo 'tmp' para nomear uma procedure temporária no DETRAN-PE?
========================================
RESPOSTA:
Não. O prefixo 'tmp' é exclusivo para tabelas temporárias (ex: tmpParcelaDebitoSefaz). Procedures devem usar o padrão convencional (ex: dbfisc01..NomeProcedure.scp).

CATEGORIA: Boas Práticas
========================================
PERGUNTA: Qual a regra correta para usar operações como A, I, E, S em nomes de procedures do DETRAN-PE?
========================================
RESPOSTA:
Use siglas maiúsculas no final do nome: A (Alterar), I (Inserir), E (Excluir), S (Selecionar). Exemplo: ConsultarTurnoHorarioA.scp (alteração de turno por exemplo).

CATEGORIA: Stored Procedures
========================================
PERGUNTA: Devo incluir um complemento em um nome de procedure como 'ObterLicencaMoto'?
========================================
RESPOSTA:
Sim. Use substantivos que respondam a 'O que selecionar? O que inserir?'. Adicione um complemento se o objetivo for insuficiente (ex: ObterLicencaMotoComplementoS.scp para consulta).

CATEGORIA: Integridade Referencial
========================================
PERGUNTA: Por que um nome de procedure com sufixo 'R' para relatório é válido?
========================================
RESPOSTA:
A operação 'R' (Relatório) é permitida quando a função não se enquadra nas operações básicas (S, I, E, A). Exemplo: EmitirDocumentoR.scp, conforme manual de nomenclatura.
